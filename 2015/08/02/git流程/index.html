    git 工作流程

  1. 下载代码，在下载的代码基础上做开发，然后提交
  2. git init 生成一个仓库，在这个生成的仓库中添加代码，推送到服务器的空仓库中

    git

  3. git push 远程仓库的地址 本地分支：远程分支

  4. git push 远程仓库的地址 本地分支

  5. git pull 远程仓库的地址 远程分支：本地分支

  6. git pull 远程仓库的地址 远程分支

  7. git clone 远程仓库的地址

  8. git clone 时会产生一个远程主机名 origin

  9. git pull 不会产生origin，但是通过git remote add origin 远程地址

  10. git pull 和git clone 的主要区别

    1. git clone的项目会自动建立origin远程仓库的别名
    2. git clone 会自动跟踪远程分支（可以直接使用git pull进行代码更新 ）
    3. 如果git pull 拉下来的代码希望建立跟踪分支，需要执行如下命令：
      • git branch –set-upstream-to=origin/master master
      • origin/master表示远程分支
      • master表示 本地分支

  11. git remote 查看远程主机的别名

  12. git remote show origin 查看远程仓库别名的具体地址

  13. 添加远程仓库的别名 git remote add origin root@180.76.188.62:/root/demo.git

  14. git fetch 从远程仓库获取分支的内容，但是不合并，之后可以通过git merge 进行合并

  15. git pull 是从远程仓库获取内容，直接合并

  1. git 本地仓库
  2. git 共享仓库
  • 具体的使用
  1. 已经存在的项目
  2. 分配git账号，或者自己生成公钥和私钥，然后把公钥提供给git管理者

  8. git stash
  9. git stash pop`

  11. 删除本地分支 git branch -d
