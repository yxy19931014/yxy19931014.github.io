gulp 使用

  1. 安装gulp,先全局安装,然后在需要构建的目录下安装gulp nmp install gulp
  2. 引入本地安装的gulp require(‘gulp’); gulp是一个对象,此对象有若干方法;引入需要的插件,比如: require(‘gulp-less’);
  3. 如果想完成 某些功能,需要使用过滤器,过滤器一般是以gulp插件的形式存在的,并且gulp插件也是基于nodejs开发的,其中有一个插件叫uglify,可以实现js代码的压缩,使用gulp-less 可以把less转换为css; 使用 gulp-cssmin可以实现css压缩
  4. 使用gulp.task(“任务名”,function(){ } ) 可以创建一个任务
  5. 在function中找到需要压缩的资源, 通过gulp.src();
  6. 使用过滤器进行加工,.pipe(过滤器名());
  7. 过滤后放到别的目录 .pipe(gulp.dest(‘./release/scripts’));
  8. 然后在cmd中运行, gulp 任务名

关系数据库

  • MySQL
  • oracle
  • DB2
  • Microsoft SQLserver
  • 所有关系型数据库都类似,所有都是表.关系.表中的字段数据,表与表之间的关系
  • 共同点:必须先有表结构才可以操作这个表
  • 而且随着业务场景的不断演变,表需要频繁的修改
  • 一个机器上安装数据库其实就是安装了一个数据服务实例
  • 一个数据服务实例上可以有多个数据库
  • 一个数据库可以有多张表
  • 一张表可以存储多条记录数据
  • 数据服务实例> 数据库> 表>记录

    非关系型数据库

  • mongodb 是非关系型数据库
  • 不需要提前设计表结构
  • 最大的好处是非常灵活
  • 存储是的json数据

    mongodb数据存储结构

  • 在一个数据服务中可以有多个数据库
  • 在一个数据库可以有多个集合(表称为集合)
  • 集合简单理解就是一个数组
  • 在一个集合中可以有多个json对象(在mongodb中成为文档),是一个特殊对象

node相关概念(node 入门)

  1. node是一个构建于chrome v8引擎之上的一个JavaScript运行时环境
  2. node可以像浏览器一样去解析和执行JavaScript代码
  3. JavaScript是一个动态脚本语言
  • 预编译
  • 再执行
  1. JavaScript之前的解析和执行一直停留在浏览器环境
  • 通过编译解析引擎:解析和执行JavaScript脚本代码
  • chrome v8 引擎是所有浏览器里面解析和执行js代码最快的
  • node作者把chrome 的v8引擎移植出来,又基于v8 引擎之上构建了一系列的API,在node中没有界面,不再需要关心HTML,css,只需关注JavaScript, 和dom ,style 等没有任何关系

angular 使用流程

Veröffentlicht am 2016-02-03

AngularJs

定义

  • AngularJS是一款由Google公司开发维护的前端MVC框架，其克服了HTML在构建应用上的诸多不足，从而降低了开发 成本提升了开发效率。
  • 使用AngularJS构建应用（App）时是以模块化（Module）的方式组织的，即将整个应用划分成若干模块，每个模块都有各自的职责，最终组合成一个整体。采用模块化的组织方式，可以最大程度的实现代码的复用，可以像搭积木一样进行开发。
  • MVC ：是一种开发模式，由模型（Model）、视图（View）、控制器（Controller）
    • 模型（Model）一般用来处理数据（读取/设置），一般指操作数据库。
    • 视图（View）一般用来展示数据，比如通过HTML展示。
    • 控制器（Controller）一般用做连接模型和视图的桥梁。
  • 具体概念
  • 做网站开发其实就是在处理一件事件（服务端返回数据，前端展示数据）
  • 服务端需要一个角色专门处理数据，成为模型
  • 需要一个角色将数据传给前端，成为控制器
  • 需要一个角色将数据呈现，成为视图

    使用

  1. 引入angular
  2. 在任意dom元素上使用一个属性ng-app就可以，ng-app 属性所在的dom元素所包含的所有子元素都属于应用的一部分
  3. 引入了angularjs之后会提供一个全局的对象angular，在此对象下有若干方法，其中angular.module()可以实例一个模块出来，var App=angular.module(‘App’,[]); // 需要两个参数 1，模块名称，2.依赖
  4. 定义控制器，（mvc，其中c指得就是controller，他是m和v的桥梁）上面步骤实例出来的模块，也是一个对象，并且在此对象下也有若干方法，其中App.controller()方法，用来创建控制器， 需要两个参数，1.名称，2.依赖,2.1 依赖的这个数组里，最后一个单元必须是一个函数

    angular 指令

  5. ng-App 指定应用根元素，至少有一个元素指定了此属性。
  6. ng-controller 指定控制器
  7. ng-show控制元素是否显示，true显示、false不显示
  8. ng-hide控制元素是否隐藏，true隐藏、false不隐藏
  9. ng-if控制元素是否“存在”，true存在、false不存在
  10. ng-src增强图片路径
  11. ng-href增强地址
  12. ng-class控制类名
  13. ng-include引入模板
  14. ng-disabled表单禁用
  15. ng-readonly表单只读
  16. ng-checked单/复选框表单选中
  17. ng-selected下拉框表单选中

    自定义指令

  • AngularJS允许根据实际业务需要自定义指令，通过angular全局对象下的directive方法实现。

数据绑定

  • AngularJS是以数据做为驱动的MVC框架，所有模型（Model）里的数据经由控制器（Controller）展示到视图（View）中。所谓数据绑定指的就是将模型（Model）中的数据与相应的视图（View）进行关联，分为单向绑定和双向绑定两种方式。

    单向绑定

  1. 将模型的数据传到视图
  • 通过双大括号和ng-bind指令来实现模型（Model）数据向视图模板（View）的绑定，模型数据通过一个内置服务$scope来提供，这个$scope是一个空对象，通过为这个对象添加属性或者方法便可以在相应的视图（View）模板里被访问。
  • 通过ng-init也可以初始化模型（Model）也就是$scope。
  1. 将视图的数据传到模型
  • 通过为表单元素添加ng-model指令实现视图（View）模板向模型（Model）数据的绑定

    双向绑定

  • 双向数据绑定

事件

  • AngularJS对事件也进行了扩展，无需显式的获取DOM元素便可以添加事件，易用性变的更强。通过在原有事件名称基础上添加ng-做为前缀，然后以属性的形式添加到相应的HTML标签上即可。如ng-click、ng-dblclick、ng-blur等。
