angularJS 粗总结

Veröffentlicht am 2016-02-03

#AngularJS

angularJS作用域

  1. 根作用域 ng-app
  2. 定义一个控制器，给元素添加一个控制器，就默认定义了一个作用域
  3. 和函数作用域类似，就近原则

    过滤器

    angular内置过滤器

  4. angularJS中用于展示数据，但是数据不符合格式要求，过滤器就是格式化展示数据
  5. 用法：在双括号中加| ，再加特定的格式名称
  6. angular 内置的过滤器

自定义过滤器

依赖注入

  1. 在现实开发中，开发人员根据需求开发自己的业务功能，这些业务功能是会‘依赖到’一些基础功能封装，例如Ajax
  2. 按上述分析，开发过程会遇到大量的依赖问题，假如不能很好解决依赖关系，开发无法进行
  3. 采用注入的方式解决依赖问题
  4. 注入是通过‘参数形式’实现的。

内置服务

  • 服务是一个对象或函数，对外提供特定的功能。

    $log 打印调试信息

  • 调试的级别
    1. notice 注意
    2. warning 警告
    3. error 错误
    4. info 提示
    5. log 打印信息 和console.log() 类似

定时器

  1. $timeout 是一个函数，延时显示
  2. $interval 是一个函数，每隔一段时间执行

$http 发异步请求

  • 客户端和服务端是两个主体，他们之间可以进行通信，但是如果是post请求，需要加特殊的请求头，来指定客户端按照特定的格式传递数据，服务端按照响应的格式接收数据。
  • PHP中$_POST 是专门用来接收formDate格式数据的，不能接收Content-Type:application/json 的,所以对应的发送请求的时候，应该加请求头：content-Type:application/X-www-form-urlencoded
  • angularjs默认的发送格式是Content-Type:application/json的，所以发送的数据应该是json格式的， $_Post不能接收这种格式的数据，可以使用file_get_contents接收
  • 如果要发送‘test=liming&age=12’,需要加Content-Type:application/X-www-form-urlencoded

jsonp（在angujs1.6版本中不用传callback：’JSON_CALLBACK‘）

  1. 前端传递一个事先定义好的函数名给服务器，然后服务器接收这个函数名并拼凑”( )”,并返回，前端就接收一个函数调用

自定义服务

  • 使用service或者factory方法

  1. factory方法

  2. service方法

模块加载

  • AngularJS在执行时是按着特定的顺序执行的，其中有两个环节（配置和运行），处在初始化之后，所以通过配置可以影响整个angularjs的执行结果
  • 服务和模块只是不同的名字，实质上是一样的，就好比函数和方法

    配置块

  • 模块的配置使用config方法，参数是一个数组

运行块

  • 一个服务不仅可以被当做依赖使用，还可以直接运行
  • 使用run方法可以直接运行一个服务
  • 使用run可以定义全局变量和方法 $rootScope是根作用域，是一个服务
  • 参数:数组

路由

  • 路由是url地址与程序的一种映射关系，通过调整地址的变化，从而调用或执行某一具体的应用程序
  • 前端的路由:AngularJS将锚点的变化封装成路由

    路由的使用

  • angular只支持一级路由
  • 可以使用路由，就意味着开发者可以根据地址的变化来执行不同的业务逻辑
  • 路由是必须经过配置后才能被使用
  • $routeParams服务可以获取路由上的参数

    路由的参数传递有两种方式

  • 一种是直接在锚点中地址后面加参数名=值，格式是?key=val&key1=val1
  • 另一种是指when中路由地址后面加/: 参数名，在锚点地址后面直接加/值
    ，格式是.when(‘/login/:name/:pass’,{templateUrl:’./views/login.html’});
    html中的格式是登陆

SPA（single page application）

  • 单页面是指功能都集中到一个页面上
  • 多页面的缺点：
    • 每次更新页面都会进行刷新
    • 不能得到很好的用户体验，页面切换的时候不能进行js操纵

      angularJS中的jQuery

  1. 在没有引入jQuery的前提下，angularJS实现了简版的jQueryLite，通过angular.element() 不能选择元素，但可以将一个DOM元素转成jQuery对象，如果引提前引入了jQuery则angular.element则完全等于jQuery。
  2. 也就是没有引入jQuery，有些jQuery中的语法不支持，引入jQuery之后，jQuery语法都支持

    bower

  3. bower 是基于nodejs的静态资源管理工具，由twitter公司开发维护，解决大型网站中静态资源的依赖问题。
  4. 依赖nodejs和git工具
  5. 安装和卸载前端所用到的工具
  6. npm install -g bower安装bower
  7. bower search 查找资源信息
  8. bower install 安装（下载）资源，通过#号可以指定版本号
  9. bower info 查看资源信息
  10. bower uninstall 卸载（删除）资源
