js高级第一天

基本类型和引用类型

  1. 基本类型：number，string，boolean，null，undefined

  2. 引用类型：object(Array,Object,Function,RegExp)

异常处理（Error）

  1. Try，throw, catch

    • Try{throw}catch(e){}

    • Try 中出现异常，直接执行catch，try中出现异常的代码后面的代码不会执行，但finally都会执行。

    • 应用场景：兼容代码try{ var xhr=new XMLHttpRequest();}catch(e){ var xhr=new ActiveXObject(Microsoft XMLHTTP)};
    • 屏蔽低级的异常信息，给用户一个友好提示

面向对象

  1. 模板—-构造函数

  2. 实例对象

    （1） 对象成员（属性和方法

    （2） 对象.属性

    （3） 对象[“属性”]

    （4）方括号中可以传变量

  3. 对象的创建

    1. 通过构造函数new

    2. 字面量 obj={username：‘张三’,gender:”male”}；object的实例相当于 var obj=new Object();obj.username=”张三”；obj.gender=”male”;

    3. 如果构造函数不需要传递参数，后面的括号可以省略

      • Var obj={}; var obj=new Object(); var obj=new Object;

      • Var arr=[];var arr=new Array();

      • Var reg=[];var reg=new RegExp();

    4. 构造函数实例化对象本质上做了什么工作

      *　开辟堆内存用来存储实例中的数据

      *　用this指向这块区域

      *　通过this向该区域中存放数据

      *　返回this

    5. 构造函数中的this指的是构造函数所创建的实例对象

    6. 构造函数中方法中的this指得是方法的调用者（实例化对象）

    7. 构造函数如果显式的返回基本数据类型，那么和不返回等效；如果返回的是引用类型的数据，那么返回的不再是this，而是return 后面的数据。

  4. 对象的本质

    • 对象的本质：无序的键值对的集合

  5. 原型

    1. 共享 prototype

    2. 构造函数.prototype.属性名 —定义

    3. 构造函数.prototype.方法 —–定义

    4. 调用 对象.共享属性名（方法）

    5. 所有的构造函数产生的实例对象都有一个属性 proto，该属性不是标准属性（浏览器中不一样，不可以在编程中使用，浏览器内部使用），该属性和构造函数中的prototype指向相同

    6. proto在实例对象中，prototype在构造函数中，指向相同

    7. 原型：减少内存消耗，实现资源共享

  6. 面向对象

    1. 面向过程的编程模式：团队开发容易产生命名冲突，不方便代码的复用

    2. 是一种编程模式，以对象的方式编程（复用性高，变量名重复概率降低）

    3. （1）构造函数中的this和原型方法中的this指向相同，都指向实例对象（2） 事件方法中的this指向的是绑定事件的对象

    4. 对象中属性的判断

  • in判断对象中是否存在某个属性，（属性在实例对象和原型对象中都可以）

    • if(“a”in window){var a=123;console.log(a);} 一定会打印123， 1.原因：全局作用域中的变量和函数都是window对象的成员。2原因：变量提升

  • hasOwnProperty（）和in的属性相似，判断属性是否在实例对象上，只能判断自定义属性，系统的属性无法判断

  • 判断一个对象是否在原型上

    • 用in判断存在，而用hasOwnProperty（）判断不存在，封装一个方法
  • 判断一个对象是否是该构造函数的实例
    • 对象.instance of(构造函数)
