      SSH

  17. SSH 是一种网络协议，用于计算机之间的加密登录(具体就是实现远程登录)

  18. 登录方式

    • 使用密码登录
      1. 首先客户端向服务器发送一个请求
      2. 服务器接收到请求之后向客户端发送自己的公钥
      3. 客户端接收到公钥后存储到指定目录
      4. 客户端输入密码然后用公钥加密，再次发送请求
      5. 服务器接收到密文后，使用自己的私钥解密然后验证通过，从而允许客户端登陆
    • 使用公钥和私钥登录（与密码没有关系）
      1. 客户端先生成自己的公钥和私钥
      2. 把客户端的公钥传递给服务器，服务器使用该公钥给客户端授权
      3. 客户端直接可以免密码登录

  19. 加密方式
    　对称加密 　非对称加密（公钥加密，私钥解密，私钥加密，公钥解密）

  20. SSH 通信过程：

    1. 客户端发送请求
    2. 服务器接收到请求之后，将服务器的公钥发送给客户端
    3. 客户端收到公钥后会存储到某个位置（c:\Users\用户名.ssh）
    4. 客户端再次发送登录请求（密码会使用公钥加密）
    5. 服务器接收到密文后，使用私钥进行解密，然后验证是否正确

注册事件的区别

  1. 方法名不同
  2. 参数不同
    • addEventListener:
      • 第一个参数是：事件类型
      • 第二个参数是：事件处理函数
      • 第三个参数是：false
    • attachEvent:
      第一个参数是事件类型
      第二个参数是事件处理函数
      没有第三个参数
  3. 事件执行的时候内部的this指向不同
    • addEventListener中的this是事件源
    • attachEvent中的this指向window
  4. 一个有on，一个没on
  5. 兼容问题：addEventListener在谷歌，火狐，ie高版本支持
    • attachEvent在ie8支持

    • 第三个参数是为true：事件捕获

    • 冒泡阶段：从里向外
    • 捕获阶段：从外向里
  • 事件有三个阶段：有一个值：1,2,3

    • 这个值是2：目标阶段

    正则表达式

  • 首先是一个表达式，可能是单个字符构成，也可能是多个字符构成，也叫规则表达式
  • 作用：是用来匹配字符串的
  • 数字、字母、汉字都是字符串类型的
  • 基本元字符：
    • . 表示的是除了\n以外的任意一个单个字符
    • [] 表示的是范围
      • [a-z]所有的小写字母中的一个
      • [A-Z]所有大写字母中的一个
      • [a-zA-Z]所有的字母中的一个
      • [0-9a-zA-Z]所有字母和数字中的一个
      • | 或者 ，在正则表达式中的优先级别最低
      • （）提升优先级别的，分组（分组：从左到右）
        • 表示的是该符号前面的第一个表达式出现了1次到多次
        • 表示的是该符号前面的第一个表达式出现了0次到多次
      • ？表示的是该符号前面的第一个表达式出现了0次到1次 另一个作用：阻止贪婪模式
      • \d 所有数字中的一个
      • \D 非数字中的一个
      • \s 空白字符
      • \S 非空白字符
      • \w 非特殊符号 包括[a-zA-Z0-9]
      • \W 特殊字符 [^a-zA-Z0-9]
      • ^ 以什么开头，在[]中表示非
      • [^] 取非
      • [.] 干掉元字符原来的意义，表示的是一个点
      • [] 就是一个\

canvas 画布

属性

  1. canvas画布默认宽高为300*150 px
  2. canvas宽高要使用canvas标签的属性设置，不能使用css设置

步骤

  1. 获取画布 var cas=document.getElementById(“cas”);
  2. 获取画笔 var ctx=cas.getContext(“2d”);
  3. 指定画图的起点 ctx.moveTo(100,100);//坐标
  4. 画路径 ctx.lineTo(300,100);//从开始的点到该点连线
  5. 连线或者填充 ctx.stroke(); 或者 ctx.fill();

    设置描边的颜色和填充的颜色

  6. ctx.strokeStyle=”red”;
  7. ctx.fillStyle=”green”;

    闭合和开辟新路径

  8. 闭合 ctx.closePath(); 将最后的点和最初的点连接起来
  9. 开辟新路径 ctx.beginPath(); 开辟新的路径
    • 如果需要设置与之前不一样的样式就需要开辟新路径
    • 如果现在填充的和之前的没有关系就需要新开辟路径
    • 开辟了新路径，必须ctx.moveTo(横坐标，纵坐标);

      填充的非零环绕定理

  10. 非零环绕原则（填充的原则）：从闭合的区域中划出一条线，这条线经过的线条中，如果正向与反向的条数相等就不填充，否则就填充（等于0不填充，非零填充）
  11. 填充的时候，如果开辟了新路径，就从新的开始填充，与之前的没有关系了。

    新属性

  12. ctx.lineCap (square,round)
  13. ctx.lineJoin (round,bevel)
