构造函数

* 抽象的模板---创建实例对象
* 实例对象
* 原型：减少内存消耗，实现资源共享
* 这三者的关系：
    1. 构造函数中都有原型属性prototype,该属性本质也是对象（object的实例对象）
    2. 原型对象中都有一个属性constructor,该属性指向原型所属的构造函数
    3. 实例对象中都有一个属性_proto_,该属性不是标准属性，不可以在编程中使用，浏览器内部使用的，该属性指向原型对象
    4. 实例对象._proto_==构造函数.prototype，结果为true

原型链

* 实例对象和原型对象之间形成的链式结构，该链式结构通过_proto_链接起来的  c1->Car.Prototype->Object.prototype->null

      SSH

  17. SSH 是一种网络协议，用于计算机之间的加密登录(具体就是实现远程登录)

  18. 登录方式

    • 使用密码登录
      1. 首先客户端向服务器发送一个请求
      2. 服务器接收到请求之后向客户端发送自己的公钥
      3. 客户端接收到公钥后存储到指定目录
      4. 客户端输入密码然后用公钥加密，再次发送请求
      5. 服务器接收到密文后，使用自己的私钥解密然后验证通过，从而允许客户端登陆
    • 使用公钥和私钥登录（与密码没有关系）
      1. 客户端先生成自己的公钥和私钥
      2. 把客户端的公钥传递给服务器，服务器使用该公钥给客户端授权
      3. 客户端直接可以免密码登录

  19. 加密方式
    　对称加密 　非对称加密（公钥加密，私钥解密，私钥加密，公钥解密）

  20. SSH 通信过程：

    1. 客户端发送请求
    2. 服务器接收到请求之后，将服务器的公钥发送给客户端
    3. 客户端收到公钥后会存储到某个位置（c:\Users\用户名.ssh）
    4. 客户端再次发送登录请求（密码会使用公钥加密）
    5. 服务器接收到密文后，使用私钥进行解密，然后验证是否正确

    6. git rm – cached 文件列表 重新把暂存区中的文件放回工作目录

    11. git reset –hard + 要回到版本的提交id(哈希文) 用本地仓库中的快照覆盖工作目录和暂存区的内容，并且放弃commitID之后的提交
