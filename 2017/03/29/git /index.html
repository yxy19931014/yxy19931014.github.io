操作计算机的几种方式

  1. 图形界面
  2. 命令行
    1. windows —> dos
    2. Linux （Unix mac）—->shell
    3. dos和shell 都有两层含义：解析器，编程脚本

      bath 命令

  • 命令格式： 命令[可选参数][参数]
  1. pwd 打开当前路径
  2. ll 显示文件详细信息 （ls -l）
  3. ls 查看当然目录下的文件只显示文件名
  4. cd 切换目录
  5. mkdir 创建目录(在当前目录下创建目录)
  6. touch 创建一个新的文件
  7. 文件中写内容的方式（echo 内容 > 文件名称）（echo 内容 >> 文件名称）[追加]
  8. cat 查看文件的内容 cat +文件名称
  9. less 查看文件内容 less+ 文件名（全屏显示）q退出
  10. tail 查看文件的结尾内容 tail -n + 数字+文件名(-n 后面的数字表示结尾的几行) tail -f abc.txt (监控文件中的内容的实时变化)
  11. rm 删除文件 rm 文件名称（不能删除目录）
  12. rmdir 删除空文件夹 rmdir + 文件夹
  13. rm -rf 删除目录（空的，有内容的都删除）
  14. mv 移动文件
  15. cp 复制文件

    vi 编辑器

  16. 三种模式：插入模式，底行模式，命令行模式（vi + 文件名【如果文件不存在，会创建一个新的文件】）
    • i，a从命令模式进入输入模式，从命令行直接保存退出shift+zz ； ctrl+f（b） 翻页; /+内容 （查找内容）；快速到文件开始处 gg ;
    • esc 退出输入模式
    • shift+： 进入末行模式，:wq +enter 退出vi编辑器 :set nu 可以显示出行号，但是不能保存到文件中，: e! 回到上一次保存的状态 :w + 新的文件名 （另存为）；shift+: +0 快速到首行，； shift:+$ 快速到末行

      SSH

  17. SSH 是一种网络协议，用于计算机之间的加密登录(具体就是实现远程登录)

  18. 登录方式

    • 使用密码登录
      1. 首先客户端向服务器发送一个请求
      2. 服务器接收到请求之后向客户端发送自己的公钥
      3. 客户端接收到公钥后存储到指定目录
      4. 客户端输入密码然后用公钥加密，再次发送请求
      5. 服务器接收到密文后，使用自己的私钥解密然后验证通过，从而允许客户端登陆
    • 使用公钥和私钥登录（与密码没有关系）
      1. 客户端先生成自己的公钥和私钥
      2. 把客户端的公钥传递给服务器，服务器使用该公钥给客户端授权
      3. 客户端直接可以免密码登录

  19. 加密方式
    　对称加密 　非对称加密（公钥加密，私钥解密，私钥加密，公钥解密）

  20. SSH 通信过程：

    1. 客户端发送请求
    2. 服务器接收到请求之后，将服务器的公钥发送给客户端
    3. 客户端收到公钥后会存储到某个位置（c:\Users\用户名.ssh）
    4. 客户端再次发送登录请求（密码会使用公钥加密）
    5. 服务器接收到密文后，使用私钥进行解密，然后验证是否正确

      git

  21. git中的三个重要区域
    1. 工作目录
    2. 暂存区
    3. 本地仓库
  22. git中文件的四种状态
    1. 未跟踪 untracked — 工作目录刚创建的文件
    2. 已暂存 staged —- 从工作目录提交到暂存区的文件
    3. 已提交 commited —– 从暂存区提交到本地仓库的文件
    4. 已修改 modified —— 暂存区修改过的文件
  23. 常用命令基本使用（本地）
    1. 建立一个放置的区域
    2. git init 初始化仓库
    3. git config –global user.name 名字
    4. git config –global user.email 邮箱
    5. git add 文件列表 将文件（未跟踪到文件）提交到暂存区（多个文件之间用空格隔开） git status 可以查看文件的状态 git -A 表示全部提交（git add * ）
    6. git rm – cached 文件列表 重新把暂存区中的文件放回工作目录
    7. git commit -m ‘ 描述信息’ —- 将暂存区的文件提交到本地仓库
    8. git log 查看历史记录
    9. git checkout –files 取消暂存区文件的修改内容
    10. git reset HEAD 使用本地仓库的最新版本覆盖暂存区，但是不覆盖工作目录，也就是取消了add，不把内容放回到工作目录
    11. git reset –hard + 要回到版本的提交id(哈希文) 用本地仓库中的快照覆盖工作目录和暂存区的内容，并且放弃commitID之后的提交

      分支

  • 从主版本拷贝若干份，修改，而不污染原来的。
  1. 创建分支 git branch 分支名称 （git branch 查看分支）
  2. 删除分支 git branch -d 分支名称
  3. 切换分支 git checkout 分支名称
  • git checkout -b 分支名称 (创建并切换分支，相当于git branch 和 git checkout 合并)
  1. 合并分支 git merge 分支名称（来源分支）
